# Video Downloader (Android, Kotlin + Jetpack Compose)

A general-purpose video downloader that works on **any website that openly
serves its video as a direct, unauthenticated file** вЂ” and intentionally
does nothing on sites that don't.

## What it does

1. You paste a page URL (or share one into the app from a browser).
2. The app loads the page's HTML and looks for video that's exposed the
   normal, open way: an HTML5 `<video src>` / `<source src>` tag, an
   `og:video` meta tag, or a direct link to an `.mp4`/`.webm`/`.mov`/`.m3u8`
   file.
3. It sends a `HEAD` request to confirm the file is genuinely public
   (no `401`/`403`, no auth headers needed).
4. If found, it downloads the file using multiple parallel byte-range
   requests (when the server supports `Accept-Ranges: bytes`) to maximize
   throughput, with a background foreground-service notification showing
   progress.

## What it deliberately does NOT do

- **No extraction from platforms whose Terms of Service prohibit
  downloading** вЂ” YouTube, Facebook, Instagram, TikTok, Twitter/X, Vimeo,
  Netflix, Prime Video, Disney+, Hulu, Hotstar, Spotify are blocked outright
  in `DownloadPolicy.kt`, regardless of whether their player happens to
  expose a technically-reachable stream URL. Add to or edit that list if
  you want to adjust policy for your own deployment.
- **No paywall or login bypass.** If a `HEAD` request comes back
  `401`/`403`, or the page requires cookies/session tokens to load the
  video, the app skips it.
- **No DRM circumvention.** Widevine/FairPlay/PlayReady-protected content
  is detected and refused.
- **No reverse-engineering of private player JavaScript** or signed-URL
  ciphers. If a site doesn't want to be scraped and doesn't expose a plain
  file link, this app won't work on it вЂ” that's by design, not a bug.

## Building

1. Open this folder in Android Studio (Koala/2024.1 or newer recommended).
2. Let Gradle sync вЂ” it will pull OkHttp, Jsoup, WorkManager, and Compose
   dependencies automatically.
3. Run on a device/emulator with API 26+.

No API keys or backend are required; everything runs on-device.

## Building on GitHub (CI)

`.github/workflows/android.yml` builds a debug APK on every push/PR and
uploads it as a workflow artifact.

**Important:** this project does not include a committed Gradle wrapper
binary (`gradlew`, `gradlew.bat`, `gradle/wrapper/gradle-wrapper.jar`) вЂ”
those are typically auto-generated by Android Studio on first open, and the
binary `.jar` isn't something to hand-write. To keep the CI build working
without that file, the workflow uses `gradle/actions/setup-gradle` to
install Gradle directly and runs `gradle assembleDebug` instead of
`./gradlew assembleDebug`.

If you'd rather use the standard `./gradlew` wrapper (recommended once
you're working locally):
1. Open the project in Android Studio once вЂ” it generates the wrapper
   files automatically, **or**
2. Run `gradle wrapper --gradle-version 8.7` from the project root if you
   have Gradle installed locally.
3. Commit the generated `gradlew`, `gradlew.bat`, and
   `gradle/wrapper/gradle-wrapper.properties` +
   `gradle/wrapper/gradle-wrapper.jar` files.
4. Then you can simplify the workflow's build step back to
   `./gradlew assembleDebug`.

## Extending it

- **Add sites to the allowed/blocked list**: edit `disallowedDomains` in
  `DownloadPolicy.kt`.
- **Support HLS (.m3u8) segment downloading**: currently `.m3u8` playlist
  URLs are detected as a "direct link" but the downloader treats them as a
  single file. To properly reassemble segmented streams you'd want to parse
  the manifest and merge `.ts` segments вЂ” happy to add that if you tell me
  which specific *open* (non-DRM) HLS sources you're targeting.
- **Download history / queue UI**: the `DownloadViewModel` currently
  supports one check-and-download flow at a time; a Room database + list
  screen would be the natural next step for a download history.

## Legal note

This app is built to only touch content that a site has made available for
direct, unauthenticated download. Downloading video from a site in a way
that violates that site's Terms of Service, bypasses a paywall, or
circumvents DRM/access controls is outside the scope of what this project
does, and you're responsible for how you deploy/modify it.
